# 03a - snow_search_team_ci

## Task type

ServiceNow Search / ServiceNow Table API task.

This is not a JavaScript task.

## Position in workflow

```text
get_alerts
  ↓
validate_interfaces
  ↓
snow_search_team
  ↓
snow_search_team_ci
  ↓
normalize_team_search
```

## Purpose

Search ServiceNow CMDB for the cluster CI name generated by `validate_interfaces` before the Team create/update payloads are normalized.

This task is lookup only. It does not create or update incidents.

## Loop configuration

Loop must be enabled.

Loop input / item list:

```text
result("validate_interfaces").teamIncidents
```

Condition:

```text
blank
```

Do not put `| length > 0` in the loop input. The loop input must be the array itself.

## ServiceNow search parameters

### Table

Use the CMDB CI table available in the ServiceNow connector.

Preferred table:

```text
cmdb_ci
```

If the connector exposes a more specific table for Cohesity clusters, use that table only if it contains the same cluster names shown in `validate_interfaces.teamIncidents[].cluster`.

### sysparm_query

Use the current loop item variable from Dynatrace:

```text
name={{ _.item.cluster }}^ORDERBYname
```

### sysparm_limit

```text
1
```

### sysparm_fields

```text
sys_id,name,sys_class_name
```

## Expected behavior

For each Team candidate:

```text
1 matching CI -> normalize_team_search uses that CI name/sys_id for cmdb_ci
0 matching CI -> normalize_team_search falls back to Cohesity (PRODUCTION)
```

## Fallback CI

If the cluster-specific CI is not found, the Team incident must use:

```text
Cohesity (PRODUCTION)
```

## Important rules

- The cluster name must come from `validate_interfaces.teamIncidents[].cluster`.
- Do not derive the Team CI from the ServiceNow incident search result.
- Do not search by IP address, interface name, alert time, node ID, or location for the Team CI.
- This CI lookup does not affect Team incident identity; incident identity remains `correlation_id`.
