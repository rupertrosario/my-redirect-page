# 03 - snow_search_team

## Task type

ServiceNow Search / ServiceNow Table API task.

This is not a JavaScript task.

## Position in workflow

```text
get_alerts
  ↓
validate_interfaces
  ↓
snow_search_team
  ↓
normalize_team_search
```

## Purpose

Search ServiceNow for an existing active Team incident for each Team correlation ID generated by `validate_interfaces`.

The search task is a lookup only. It does not create or update incidents.

## Loop configuration

Loop must be enabled.

Loop input / item list:

```text
result("validate_interfaces").teamIncidents
```

Condition:

```text
blank
```

Do not put `| length > 0` in the loop input. The loop input must be the array itself.

## ServiceNow search parameters

### sysparm_query

Use the current loop item variable from Dynatrace:

```text
correlation_id={{ _.item.correlation_id }}^stateNOT IN6,7^ORDERBYDESCsys_updated_on
```

### sysparm_limit

```text
2
```

### sysparm_fields

```text
sys_id,number,state,short_description,correlation_id,sys_updated_on
```

## Expected behavior

For each Team item:

```text
0 matching active incidents → normalize_team_search.createTeamIncidents[]
1 matching active incident  → normalize_team_search.updateTeamIncidents[]
2 matching active incidents → normalize_team_search.noWriteTeamIncidents[]
```

## Important rules

Do not search by IP address, interface name, alert time, node ID, or location for Team incidents.

Team incident identity is the cluster-level correlation ID:

```text
cohesity_ifdown_team_<cluster_id>
```

The IP/interface details belong in the comment/description, not in the ServiceNow search identity.
